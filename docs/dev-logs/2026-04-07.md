## 新增：多层模型 Fallback 架构

### 背景
Gemini API 存在不稳定风险（限流、宕机），需要备用方案保障服务连续性。

### 实现
在后端 `/api/gemini/generate` 路由实现 3 层 fallback 链：

| 优先级 | Provider | Key 来源 | 适用用户 |
|--------|----------|---------|---------|
| 1 | Gemini 2.5 Flash | Secret Manager | 全部 |
| 2 | OpenRouter (google/gemini-2.5-flash) | Secret Manager | Pro only |
| 3 | OpenRouter (备用账号) | 环境变量 | Pro only |

- 每层失败自动降级到下一层
- 响应中包含 `provider` 字段标明实际使用的 provider
- 免费用户仅走 Gemini（控制成本）

### 文件变更
- `backend/src/config.ts` — 新增 OpenRouter 配置项
- `backend/src/routes/gemini.ts` — 重构为 provider 架构，fallback 逻辑
- `backend/.env.example` — 补全新环境变量

## 修复：Fallback Prompt 未生成证件照（重大 Bug）

### 问题
OpenRouter fallback 生成的图片没有变成证件照，仅返回原图或文本分析结果。

### 根因
- Gemini 直连 API 使用 `responseModalities: ["TEXT", "IMAGE"]` 明确要求图片输出
- OpenRouter 使用 OpenAI 兼容的 chat API，缺少该参数，模型默认返回文本

### 修复
1. 添加 `modalities: ["text", "image"]` 参数请求图片输出
2. 添加 system prompt 明确角色为"专业证件照编辑器"
3. 包装 user prompt，强调这是图片编辑任务，要求返回编辑后的图片而非文本

## 成本优化

### P0：降低 Pro 日生成上限
- `UsageManager.swift`: Pro 用户日上限从 20 次降至 10 次
- 防止重度用户导致亏损（月订阅 JP¥600，满用 600 次时单次摊收入仅 JP¥1，低于 API 成本）

### P0：免费用户不触发高成本 Fallback
- `gemini.ts`: provider 新增 `proOnly` 标记
- `tier !== "pro"` 时跳过 OpenRouter，仅走 Gemini 直连

### 统一模型降成本
- Pro 端点从 `gemini-3.1-flash-image-preview` 改为 `gemini-2.5-flash-image`
- Pro 用户差异变为：高分辨率(1024px) + fallback 保障

### 移除 OpenAI Fallback
- 单次成本过高（JP¥7.5-15），不适合当前收入规模
- 从 config 和 fallback 链中完全移除
