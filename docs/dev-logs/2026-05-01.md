## 修复：证件照头部比例不符合国标

### 问题
生成的居民身份证照片人物到腰部，不符合中国证件照标准——标准要求头部占照片高度约 2/3，仅露头颈到衣领。

### 根因
- `preserveSuffix` 包含"不要裁剪或改变画面构图比例"，阻止 AI 重新构图
- 各规格 prompt 缺少明确的头部比例指令
- HivisionIDPhotos `head_measure_ratio` 使用默认值（约 0.2），头部过小

### 修复

**`ios/AIIDPhoto/Models/IDPhotoSpec.swift`**
- 新增 `idFramingSuffix`，基于 GA461-2004 国标：
  - 头顶到下巴占照片高度约 2/3
  - 仅包含头部、颈部到衣领，不露肩膀以下
  - 头顶距顶边约 5% 空白，人脸水平居中
- 重写全部 15 个规格 prompt，各规格按实际标准定制：
  - 居民身份证 / 一寸 / 二寸 / 驾驶证 / 学生证 / 社保卡 / 大一寸 / 小二寸 / NCRE → `idFramingSuffix`
  - 护照 → 头部占 65%（MFA 标准，略宽松）
  - 简历照 → 露肩膀，半身构图
  - 半身照 / 全身照 → 不限构图
- `preserveSuffix` 移除构图锁定短语，仅保留面部五官/发型不变约束

**`backend/src/routes/gemini.ts`**
- `head_measure_ratio` 从默认值改为 `0.35`（头部占 ~2/3 照片高度）

## 修复：打印排版预览图片被压扁

### 问题
打印排版预览中每张证件照高度被压缩，照片变形。

### 根因
AI 生成图片的宽高比不一定等于目标规格，原代码使用 `gc.draw(cgImage, in: rect)` 拉伸填充，导致变形。

### 修复
**`ios/AIIDPhoto/Services/PrintLayoutService.swift`** — 新增 `drawAspectFill()`：
- 使用 `max(scaleX, scaleY)` 保持宽高比缩放（scale-to-fill）
- `context.clip(to: rect)` 裁剪超出部分
- 图片居中放置在目标格子内
- 替换所有调用点，消除拉伸变形

## 安全加固

对 `/api/gemini/generate` 端点新增两项输入校验（**`backend/src/routes/gemini.ts`**）：

| 参数 | 校验规则 | 返回码 |
|------|---------|--------|
| `specBgColor` | 必须匹配 `[0-9a-fA-F]{6}` | 400 |
| `specWidth` / `specHeight` | 100–2000 像素范围内 | 400 |

两项校验仅在 `hasSpec = true` 时触发，不影响不带规格参数的普通请求。
