# 开发日志 — 2026-07-20

## 摘要

修复微信小程序**相册选图与拍照上传完全失效**的问题。根因是 `manifest.json` 里
`mp-weixin.__usePrivacyCheck__` 被设为 `false`，与已在微信后台审核通过的《用户
隐私保护指引》相互矛盾：微信自 2023-10-17 起强制启用隐私授权流程，而这个开关为
`false` 时，代码里已经写好的自定义隐私授权状态机（`wx.onNeedPrivacyAuthorization`
/ `wx.getPrivacySetting` / `agreePrivacyAuthorization` 按钮）根本不会被微信触发，
导致用户点上传/拍照时接口被判定"需要授权但无人处理"而直接失败。

将开关改为 `true` 后，整套流程闭环：首次点上传/拍照 → 触发自定义隐私弹窗
（`PrivacyConsentDialog`）→ 用户点"同意并继续"（`open-type="agreePrivacyAuthorization"`）
→ 微信记录授权并放行 → 相册/相机正常打开。配套给 `creation.vue` 的 `chooseImage`
失败分支补了错误日志、并区分"用户主动取消"（不再误弹错误提示）。

已用 `miniprogram-ci`（`npm run upload:mp-weixin`）上传 **1.0.4 开发版**至微信后台。
真机端到端验证（体验版实机测隐私弹窗）待后续在微信公众平台完成。
