- 逐一对账前端每个网络调用与后端路由：baseURL、鉴权（Bearer / X-App-Key）、

### 2. 修复 generate 永久挂起（504 根因）⚠️ 最关键
- 根因：为「客户端超时退还次数」引入的 `req.on("close")` 检测，在**正常请求**里
  就提前触发（请求体被消费后即 emit close），导致 `commitAndSend` 跳过发送响应，
  **所有生成请求永不返回 → 全部 504 / 客户端超时**。
- 定位手段：服务器本地直打 `127.0.0.1:9528`（绕过 nginx+GFW）仍挂死 + 错误日志里
  9 条「Client disconnected before delivery」误判退款（含 basic 请求）。
- 修复：移除该断开检测，恢复「直接发送 + 消费次数」的原始可用行为。
- 验证：本地 13.6s、公网经 nginx+TLS 18.7s 正常出图。

### 3. 接通第二段 AI 形象照（Hivision → 百炼）
- 现象：用户「传生活照、出生活照」。实测证明 Hivision 基础抠图/换底/裁切**正常工作**
  （/idphoto 直测 3.4s），但第二段 AI 换装/美颜**从不触发**——因前端把外观指令拼进
  `prompt`（Hivision 忽略 prompt），从未作为独立 `cosmeticPrompt` 发送。
- 前端 `api/gemini.js`：将换装/发型/配饰/美颜合成独立 `cosmeticPrompt` 发送，
  强制「保持人物五官/身份/肤色不变」防证件照失真。
- 实测：生活照 → 深色西装白底职业照，人脸一致，~16s。男/女/偏心人脸三例均成功。

### 4. Rule A — 免费档只开放基础功能
- 终身免费 3 次只走 Hivision 基础证件照，**不调用任何付费 AI**（美颜/换装/兜底）；
  付费次数（`reservedAttempt==="paid"`）才进第二段。
- 堵住「免费请求构造 cosmeticPrompt 触发付费美颜」「免费 Hivision 失败兜底到付费
  百炼」两处成本泄漏。免费档失败即退还次数。

### 5. 第二段多模型备份 + 延迟收敛
- 第二段改为**按序尝试多个百炼模型互为备份**：
  - Pro：`qwen-image-edit-plus` → `qwen-image-edit` → `wanx2.1-imageedit`
  - 普通：`wanx2.1-imageedit` → `qwen-image-edit`
- 受总时限 `COSMETIC_DEADLINE_MS=110s` 约束，按剩余时间收紧单次超时，超时即降级
  Hivision 基础图（保证有图、不挂死）。
- provider 调用超时 90s→70s；`callQwenImageEdit` / `callBailian` 支持传入超时。
- **给 provider 结果图（OSS URL）下载加 25s 超时**：此前无超时，OSS 偶发慢会无限
  挂起并耗尽 undici 连接池拖垮后续请求（504 帮凶）。

### 6. 其余稳健性修复
- 当日预算改为「调用 provider 前才计数」，无效/被拒请求不再消耗额度（防刷）。
- `manifest.json` 配置 `networkTimeout=150s`，与前端超时对齐。

### 7. 接入 miniprogram-ci 一键上传
- `scripts/upload-wechat.mjs`：免开发者工具上传 `dist/build/mp-weixin`，自动读
  appid/版本/备注；私钥查找优先级 环境变量 > `~/.config/foyli/wechat/<appid>.key`
  （repo 外安全目录）> 仓库内兜底。
- npm 脚本 `upload:mp-weixin` / `release:mp-weixin`，**强制走 IPv4**
  （`--dns-result-order=ipv4first`，微信 IP 白名单仅支持 IPv4）。
- 密钥安全：移至 `~/.config/foyli/wechat/`（权限 600），根 `.gitignore` 防 `private.*.key` 入库。
- 已上传：1.0.0 → 1.0.2（开发版，robot 1，主包 ~460KB）。
