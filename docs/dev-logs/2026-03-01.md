### 3. Phase 1 — Bug 修复

**SubscriptionManager 逻辑 Bug**
- 问题：`updateEntitlements()` 中 `for await in Transaction.updates` 是无限循环，导致后面的 `Transaction.currentEntitlements` 检查永远执行不到
- 修复：拆分为 `checkCurrentEntitlements()`（启动一次性检查）和 `listenForUpdates()`（长期监听），分别用独立 Task 启动

**GeminiError + Codable 重构**
- 实现 `LocalizedError`，4 个 case 均有中文用户可读错误信息
- 新增 `invalidImage` case
- 所有 JSON 解析从 `JSONSerialization` 重构为 `Codable` 模型（`GeminiRequest`/`GeminiResponse`/`BackendGenerateRequest`/`BackendGenerateResponse`）

### 4. GCP Cloud Run 后端代理搭建

**GCP 配置：**
- 新建项目 `ai-id-photo-prod`（Firebase 付款账号）
- 启用 API：Cloud Run、Secret Manager、Cloud Build、Artifact Registry、Gemini
- 创建 Gemini API Key，限定权限到 `generativelanguage.googleapis.com`
- Key 存入 Secret Manager，Cloud Run SA 只读权限

**Firebase：**
- `firebase projects:addfirebase ai-id-photo-prod` 接入 Firebase

**后端服务（`backend/`）：**
- Express + TypeScript，`POST /api/gemini/generate`
- IP 速率限制 10次/分钟
- Dockerfile 多阶段构建

**部署过程：**

- 后续推荐直接 `--source` 方式

## 技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 后端技术栈 | GCP Cloud Run + Node.js/TypeScript | 用户熟悉 GCP 生态 |
| 部署区域 | `asia-northeast1`（东京） | 用户主要市场 |
| API Key 管理 | Secret Manager | GCP 原生，与 Cloud Run 集成最简单 |
| Firebase | 接入但暂不使用 | 为未来 Auth/Analytics 做准备 |

## 问题 & 解决方案

| 问题 | 原因 | 解决 |
|------|------|------|
| Cloud Build 部署失败 | compute SA 缺少 Storage 权限 | 首次改用本地 Docker 构建 |
| 模拟器 App 启动失败 | Bundle ID 写死为占位符 | 用 `xcrun simctl listapps` 查到真实 ID |
| Firebase 认证失效 | Token 过期 | `firebase login --reauth` |
| "Wei" 计费账号超额 | 项目太多 | 改用 "Firebase 付款" 账号 |
