# 光影形象馆 CN — 成本收益模型

> 更新日期：2026-05-24
> 状态：待重算。旧版模型基于 GCP Cloud Run + Gemini / OpenRouter 成本，已经不适合作为 CN 版上线或投放决策依据。

---

## 当前结论

CN 版后端已经迁到阿里云 ECS，主链路也调整为 HivisionIDPhotos + 阿里云百炼/Qwen/Wanx。因此，上线前成本模型必须按真实账单和当前产品策略重新计算。

目前可以保留的判断：

- 当前更适合按张收费，不建议订阅制；多数用户只会在证件照/简历照场景使用一次。
- 每张成片应限制生成尝试次数，避免“无限重试”放大 AI 成本。
- 上线早期不建议大规模投放，先用 TestFlight/小流量验证单次生成成本、成功率和转化率。

目前不能沿用的旧判断：

- 不能再按 Gemini 单次 token 成本估算图片生成成本。
- 不能再按 GCP Cloud Run / Secret Manager 固定成本估算后端成本。
- 不能再假设 512px / 1024px 成本差异等同于 Gemini 输出 token 差异。

## 需要补齐的数据

| 数据 | 来源 | 用途 |
|------|------|------|
| ECS 实际月成本 | 阿里云账单 | 固定成本 |
| 流量 / 带宽 / CDN 成本 | 阿里云账单 | 下载与图片传输成本 |
| Hivision 单次成本 | Hivision 服务账单或自建资源成本 | 主证件照生成成本 |
| 百炼/Qwen/Wanx 单次成本 | 阿里云百炼账单 | 外观编辑和 fallback 成本 |
| 生成成功率 | 后端日志 | 真实有效成本 |
| 平均重试次数 | 后端日志 + iOS 埋点 | 成本放大系数 |
| 单张成片购买转化率 | App Store / StoreKit 数据 | 估算收入 |
| 单张成片退款率 | App Store Connect | 净收入估算 |

## 建议的最小模型

### 单次生成成本

```text
单次有效生成成本 =
  (Hivision 成本 + cosmetic pass 成本 + fallback 成本 + 带宽成本)
  × 平均尝试次数
  ÷ 成功率
```

其中：

- 无外观编辑：主要成本来自 Hivision。
- 有外观编辑：Hivision 成功后追加百炼/Qwen/Wanx 成本。
- Hivision 失败：会进入 prompt-based fallback，成本和质量都需要单独统计。

### 单张成片损益

```text
单张成片净利润 =
  App Store 税后单张收入
  - 3次生成机会 × 单次有效生成成本
  - 平均退款/失败补偿成本
```

建议先按两个价格档测算：

| 档位 | 假设 |
|------|------|
| 优惠价 | ¥3.80/张，验证首单付费和审核通过率 |
| 常规价 | ¥9.90/张，作为长期基准价 |

上线前需要确认：3 次生成机会是否足够覆盖“不满意重修”，同时仍能保证优惠价不亏损。

## 上线前成本保护

| 项目 | 当前建议 |
|------|----------|
| 后端总预算 | 保留 `DAILY_BUDGET` 熔断，按真实单次成本重新设置 |
| App 调用密钥 | 阿里云正式切流前将 `REQUIRE_APP_KEY=true` |
| 成片制作包 | 每包 3 次生成机会，成功才扣次数 |
| 无付费用户 | 直接展示购买页；推荐奖励可作为小范围拉新工具 |
| 失败重试 | iOS 端保守重试，后端记录 provider 和失败原因 |
| 日报 | 上线首周每天核对请求量、成功率、provider 分布和账单 |
