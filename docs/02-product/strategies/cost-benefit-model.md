## 一、基础数据

### 定价结构

| 项目 | 价格 | 说明 |
|------|------|------|
| Pro 月付 | $1.99/月 | 无免费试用，即时付费 |
| Pro 年付 | $12.99/年 | 无免费试用，相当于 $1.08/月 |
| Apple 佣金 | 15% | Small Business Program（年收入 < $1M） |

### 单位成本（分层定价）

| 用户层级 | API 单价 | 分辨率 | 备注 |
|----------|---------|--------|------|
| 免费用户 | **~$0.027/次** | 512px | 输入降采样 → 更少 tokens（~747 output tokens × $60/M） |
| Pro 用户 | ~$0.05/次 | 1024px | 全分辨率，适合打印 |

| 其他成本项 | 单价 | 备注 |
|-----------|------|------|
| Cloud Run（后端代理） | ~$0.000005/请求 | 冷启动 + 512Mi，极低 |
| AdMob 激励广告收入 | +$0.0174/次 | 日本市场 eCPM ≈ $17.35 |

### 免费用户成本覆盖率（核心变化）

| 指标 | 方案 A（512px） | 旧方案（1024px） |
|------|---------------|----------------|
| API 成本/次 | $0.027 | $0.05 |
| 广告收入/次 | $0.0174 | $0.0174 |
| 缺口/次 | **-$0.010** | -$0.033 |
| 广告覆盖率 | **64%** | 35% |

> 512px 策略将免费用户亏损从 $0.033/次降到 $0.010/次，广告覆盖率接近翻倍。

### 固定成本

| 项目 | 年费 |
|------|------|
| Apple Developer Program | $99 |
| GCP 最低消费（Cloud Run + Secret Manager） | ~$5/月 = $60/年 |
| 域名 & 隐私政策页面 | ~$15/年 |
| **固定成本合计** | **~$174/年** |

## 二、用户行为假设

### 免费用户

| 指标 | 数值 | 依据 |
|------|------|------|
| 首次免费生成 | 1 次 | 产品设计 |
| 看广告生成 | 平均 1.5 次/用户生命周期 | 证件照是低频刚需，用完即走 |
| 总生成次数 | 2.5 次/用户 | |
| 广告观看 | 1.5 次/用户 | 首次免费，之后看广告 |

### Pro 用户

| 指标 | 数值 | 依据 |
|------|------|------|
| 活跃生成天数 | 首月 5 天，之后 1-2 天/月 | 证件照场景集中 |
| 每活跃日生成次数 | 4 次 | 试不同规格/背景/美颜 |
| 首月总生成 | 20 次 | |
| 后续月均生成 | 6 次 | |
| 平均订阅时长 | 2.5 个月（月付）/ 12 个月（年付） | 工具类 app churn rate 高 |

### 转化率

| 指标 | 保守 | 中等 | 乐观 |
|------|------|------|------|
| 免费 → Pro 转化率 | 3% | 5% | 8% |
| 月付 : 年付比例 | 60:40 | 50:50 | 40:60 |
| 月留存率（月付用户） | 40% | 50% | 60% |

> **注意**：取消免费试用后，转化率可能略有下降（预估 -0.5~1%），但 ARPU 提升（无 0 收入试用用户），实际不确定，需上线后 A/B 测试验证。

## 三、单用户经济模型

### 免费用户（微亏，可控）

| 项目 | 金额 |
|------|------|
| API 成本（512px） | 2.5 次 × $0.027 = -$0.068 |
| 广告收入 | 1.5 次 × $0.0174 = +$0.026 |
| **净损益** | **-$0.042/用户** |

> 对比旧方案 -$0.099/用户，亏损减少 **57%**。

### Pro 月付用户（盈利）

| 项目 | 金额（2.5 个月生命周期） |
|------|------|
| 订阅收入 | 2.5 × $1.99 × 85% = +$4.23 |
| API 成本（1024px） | (20 + 1.5×6) × $0.05 = -$1.45 |
| **净利润** | **+$2.78/用户** |
| **LTV** | **$4.23** |

### Pro 年付用户（盈利）

| 项目 | 金额（12 个月） |
|------|------|
| 订阅收入 | $12.99 × 85% = +$11.04 |
| API 成本（1024px） | (20 + 11×6) × $0.05 = -$4.30 |
| **净利润** | **+$6.74/用户** |
| **LTV** | **$11.04** |

### 混合 LTV（按 50:50 月/年付比）

**每个 Pro 用户平均 LTV ≈ $7.64，净利润 ≈ $4.76**

## 七、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| Gemini API 涨价 | 成本翻倍 | 缓存相同规格结果 / 切换到更便宜模型 |
| 竞品涌入 | 转化率下降 | 深耕日本市场（便利店打印闭环是壁垒） |
| App Store 审核拒绝 | 延迟上线 | 提前准备隐私政策、人脸数据说明 |
| 低转化率 (<2%) | 无法盈利 | 调整定价 / paywall 位置优化 / 恢复免费试用 |
| 取消试用后转化下降 | 转化率降 1% | 用 512px vs 1024px 画质差异推动升级 |
| 汇率波动（JPY/USD） | 日本用户实际支付感知 | 按日元定价（¥300/月，¥1,800/年） |

## 八、核心竞争壁垒

1. **便利店打印闭环**：L判/2L判排版 → 7-Eleven 直接打印，竞品很少做到
2. **多国规格专业性**：在留卡、マイナンバー等小众但刚需的规格
3. **极低定价**：$1.99/月（照相馆拍一次 ¥1,000+，证件照机 ¥800+）
4. **4 语言本地化**：覆盖在日外国人（中/韩/英），这是一个被忽略的大市场
5. **分辨率差异化**：免费 512px（手机查看够用）vs Pro 1024px（打印级品质），自然推动升级
