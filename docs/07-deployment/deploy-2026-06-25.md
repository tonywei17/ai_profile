## 后端部署流程（本次执行）

```bash
# 1. 本地编译 + 打包（仅 dist + src，不碰 data/certs/.env）
cd backend && npm run build
tar czf /tmp/aiidphoto-deploy.tgz dist src

# 2. 上传（sshpass scp）
sshpass -p '***' scp /tmp/aiidphoto-deploy.tgz root@8.148.158.232:/tmp/

# 3. 服务器侧：备份 → 解包 → 重启
cd /opt/aiidphoto-backend
cp -r dist backups/dist-$(date +%Y%m%d-%H%M%S)   # 每次部署都备份，可回滚
cp -r src  backups/src-$(date +%Y%m%d-%H%M%S)
tar xzf /tmp/aiidphoto-deploy.tgz -C /opt/aiidphoto-backend
pm2 restart aiidphoto-backend --update-env

# 4. 验证
curl https://aiphoto-cn.foyli.cloud/health           # {"status":"ok"}
# 本地直打 127.0.0.1:9528/api/gemini/generate 验证出图
```

> 注意：`data/`（payment-store.json 支付/次数数据）、`certs/`（微信支付证书）、
> `.env`、`node_modules`、`ecosystem.config.cjs` **均不在上传包内**，绝不覆盖。

### 回滚

```bash
cd /opt/aiidphoto-backend
cp -r backups/dist-<TS> dist && pm2 restart aiidphoto-backend
```
