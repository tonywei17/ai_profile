## 7. 通过的项（已审计为符合标准）

- backend 构建干净（`tsc` 无报错）
- `apiKeyAuth` middleware 在 `REQUIRE_APP_KEY=true` 时正确返回 401，跳过 `/health` 与 `/legal`
- `rateLimit`（10/min/IP）+ `generateLimiter`（3/min/IP）+ `dailyBudget` 三层保护齐全
- `helmet` + `cors({ origin: false })` + `trust proxy` 配置合理
- HivisionIDPhotos 两阶段流水线（`/idphoto` → `/add_background`）+ Bailian fallback 链路结构清晰
- base64 解码 robust：去 `data:` 前缀、whitespace、URL-safe、补 padding 全覆盖
- `requestLogger` 不打 base64 body，结构化 JSON 配 severity 字段
- SubscriptionManager `Transaction.updates` listener 正确，消耗品 `transaction.finish()` 调用到位
- Camera / PhotoLibrary 权限文案中文化，Info.plist 通过 `plutil -lint`
- KeychainHelper 使用 `kSecAttrAccessibleWhenUnlockedThisDeviceOnly`
- 生成成功后才扣次数；失败路径需补真实后端错误场景验证

## 8. 安全 / 成本 / 性能备注

- 本轮触发了 1 次真实阿里云后端生成，产生 1 次实际模型成本
