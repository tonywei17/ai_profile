# 微信公众平台隐私指引填写清单

以下内容用于微信公众平台「设置与开发 → 服务内容声明 → 用户隐私保护指引」。应按后台实际字段逐项选择和填写，不要声明代码未使用的手机号、位置、通讯录等信息。

## 开发者处理的信息

| 信息或权限 | 使用目的建议文案 |
| --- | --- |
| 选中的照片或视频信息 | 用于取得用户主动选择的照片，生成证件照、职业形象照及打印排版图。 |
| 摄像头 | 用于用户主动拍摄照片并生成证件照或职业形象照。 |
| 写入相册 | 用于在用户主动点击保存后，将生成照片或打印排版图保存到系统相册。 |
| OpenID 对应标识 | 用于识别小程序用户、登录服务、同步免费及已购生成次数，并防止权益错发。 |
| 订单和交易信息 | 用于创建和查询虚拟支付订单、发放生成次数、退款及处理交易问题。 |
| IP 地址、请求时间、请求路径、状态码、请求 ID | 用于接口安全、限速、防止滥用、故障定位和服务稳定性保障；服务器日志不记录照片内容。 |
| 人像照片及面部特征 | 用于人像检测、抠图、裁切、背景替换和外观编辑，仅处理用户主动上传或拍摄的照片。 |
| AI 生成内容标识和导出记录 | 用于为生成图片写入隐式标识，并在用户主动申请导出无显式水印图片时记录生成内容 ID、用户标识、导出类型和时间；日志不包含照片内容，保存期限不少于六个月。 |

## 敏感个人信息说明

建议用途说明：

> 用户主动上传或拍摄的人像照片可能包含面部特征，属于敏感个人信息。处理该信息是完成人像检测、抠图、裁切、换底色和外观编辑所必需。照片通过 HTTPS 传输，其中 HivisionIDPhotos 当前部署在中国大陆以外的云区域，会发生完成本次处理所必需的跨境传输；照片不用于广告画像、公开展示、出售或模型训练，我们的服务器不作持久化保存。

建议必要性说明：

> 若不处理用户选择的人像照片，将无法提供证件照和职业形象照生成功能；用户可拒绝授权或不上传照片，并仍可浏览无需照片处理的页面。

## 第三方信息共享清单

### 微信小程序基础能力

- 第三方主体：深圳市腾讯计算机系统有限公司。
- 共享信息：登录临时凭证、OpenID 对应标识、用户主动选择或拍摄的照片、相册保存结果。
- 使用目的：微信登录、照片选择、相机拍摄及保存图片。
- 使用场景：用户登录、主动选择/拍摄照片、主动保存生成结果时。
- 共享方式：调用微信小程序系统接口。

### 微信小程序虚拟支付

- 第三方主体：财付通支付科技有限公司及腾讯相关主体。
- 共享信息：商品 ID、业务订单号、支付状态、交易单号、退款及投诉信息。
- 使用目的：完成虚拟商品支付、发货、退款和交易管理。
- 使用场景：用户主动购买生成次数时。
- 共享方式：调用微信小程序虚拟支付能力。

### 阿里云 ECS

- 第三方主体：阿里云计算有限公司。
- 共享信息：照片请求、生成参数、IP 地址及必要请求日志。
- 使用目的：提供后端服务器、HTTPS 接口、安全控制和请求转发。
- 使用场景：用户登录、查询权益或提交照片生成请求时。
- 共享方式：通过 HTTPS 传输至运营者部署在阿里云的服务器。

### 阿里云百炼 / 通义万相

- 第三方主体：阿里云计算有限公司。
- 共享信息：用户主动上传的照片、文本编辑指令和生成参数。
- 使用目的：外观编辑、换装、美颜、发型及背景等图像处理。
- 使用场景：用户主动选择对应 AI 图像编辑功能时。
- 共享方式：由运营者后端通过加密接口传输。

### HivisionIDPhotos

- 第三方主体：运营者部署的 HivisionIDPhotos 图像处理服务及其 Google Cloud 基础设施提供方。
- 共享信息：用户主动上传的照片、目标尺寸和背景颜色。
- 使用目的：人像检测、抠图、裁切和证件照底色生成。
- 使用场景：用户提交证件照生成请求时。
- 共享方式：由运营者后端通过 HTTPS 传输至部署在中国大陆以外云区域的图像处理服务，存在必要的个人信息跨境处理。

> 后台第三方共享清单如支持分别填写基础设施提供方，应单独增加 Google Cloud；不要仅填写“HivisionIDPhotos 开源项目”，因为开源项目本身不是实际接收数据的主体。

## 不应勾选

- 手机号码、通讯录、短信、精确位置、运动数据、麦克风、蓝牙、身份证号码、银行卡号码。
- 若后台没有单独的 OpenID、订单或日志选项，可在「其他信息」中分别填写，不要合并成人脸信息。
